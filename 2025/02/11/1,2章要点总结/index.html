参数化抽象和规格化抽象

1. 参数化抽象（Parameterized Abstraction）

定义

参数化抽象通过引入参数（如泛型、方法参数化等）提高代码复用性和灵活性。

示例 2：方法参数化

2. 规格化抽象（Specification Abstraction）

定义

规格化抽象通过接口（Interface）或抽象类（Abstract Class）定义行为规范，而不关心具体实现。

示例 1：使用接口（Interface）

示例 2：使用抽象类（Abstract Class）

个人理解

参数化抽象用参数替换数据特征，规格化抽象则是将过程细节抽象成所需行为（即“能干什么”）。

数据抽象是面向对象程序设计的基础

  • Java程序由类和接口构成，类和接口主要由一系列方法定义而成。
  • 类用于定义过程集合以及定义新的数据类型
  • 类和接口组成了包。包作为封装机制，以及提供了包内共享信息的途径和防止外部使用

不导入（import）包也可以用其完全名称调用，即包的等级末端加上其需要调用的类名：如mathRoutines.Num。

  • 每个变量都有一个表明其类型的声明。除了八种原始类型，所有其他类型都是对象。一些对象类型会在包内提供，比如java.lang包提供String类型。
类型 == 比较内容
基本数据类型 (int, double…) 数值是否相等
引用类型 (String, List…) 内存地址是否相等

可变性

不可变对象（如String）任何对其的修改，实际上都会创建一个新的对象，原来的对象仍然存在，如果没有其他变量引用它，它会被垃圾回收。

可变对象的状态可以在其生命周期内发生变化。例如，ArrayList、HashMap、StringBuilder 都是可变的。

Java 中的显示类型和实际类型分析

在 Java 中，对象的 显示类型（Declared Type） 和 实际类型（Runtime Type） 是不同的概念，尤其在继承和多态的场景下，这两者的区别尤为重要。

1. 关键概念

类型 定义 决定什么
显示类型（Declared Type） 变量在声明时的类型 编译期方法调用、类型检查
实际类型（Runtime Type） 变量运行时的对象类型 运行时方法执行的版本

类型层次

  • Java 的类型系统是一个层次结构，Object 是所有类的根类。
  • 子类型继承了父类型的所有方法，并且可以扩展或重写这些方法。
  • 显示类型和实际类型的关系：
    • 显示类型是编译时已知的类型。
    • 实际类型是运行时对象的真实类型。
    • Java 确保显示类型是实际类型的父类型或相同类型。

深入分析

  • 多态性：通过子类型和父类型的关系，Java 实现了多态性。这使得代码可以更通用，能够处理多种类型的对象。
  • 类型安全：强制转换提供了在运行时检查类型安全性的机制，但过度使用可能导致运行时错误。
  • 设计原则：理解类型层次和继承关系有助于设计更灵活和可扩展的面向对象系统。
